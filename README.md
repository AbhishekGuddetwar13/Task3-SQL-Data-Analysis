# Task 3 – SQL for Data Analysis

## Objective

The objective of this task was to use SQL queries to extract, analyze, and summarize data from an E-commerce database. The task focused on applying SQL concepts such as filtering, aggregation, grouping, views, and handling null values.

## Tools Used

* MySQL Workbench
* SQL
* E-commerce Dataset

## Database Creation

A database named `ecommerce_db` was created and an `ecommerce_data` table was designed with the following fields:

* InvoiceNo
* StockCode
* Description
* Quantity
* InvoiceDate
* UnitPrice
* CustomerID
* Country

## SQL Operations Performed

### 1. Data Retrieval

Retrieved all records from the dataset using the SELECT statement.

### 2. Total Revenue Analysis

Calculated total revenue using:

Revenue = Quantity × UnitPrice

This helped determine the overall sales generated by the business.

### 3. Top Customers Analysis

Identified the top 10 customers based on revenue contribution. This analysis highlighted the most valuable customers.

### 4. Average Revenue Per User (ARPU)

Calculated the average revenue generated per customer using a subquery and aggregate functions.

### 5. Country-wise Order Analysis

Filtered orders from the United Kingdom using the WHERE clause.

### 6. Customer Sales View

Created a SQL View named `customer_sales` to store customer-wise revenue information for future analysis.

### 7. Revenue by Country

Analyzed revenue generated from different countries using GROUP BY and aggregate functions.

### 8. Monthly Sales Analysis

Calculated revenue month-wise to identify sales trends and seasonality.

### 9. Handling Missing Values

Identified records with missing CustomerID values using the IS NULL condition.

## SQL Concepts Demonstrated

* SELECT
* WHERE
* GROUP BY
* ORDER BY
* Aggregate Functions (SUM, AVG)
* Subqueries
* Views
* NULL Handling

## Key Learnings

* Learned how to create databases and tables.
* Practiced importing and analyzing structured data.
* Gained experience with aggregate functions and grouping.
* Understood how to create reusable views.
* Learned methods to identify missing values in a dataset.
* Improved SQL query writing and data analysis skills.

## Conclusion

This task provided hands-on experience in SQL-based data analysis. Various SQL operations were performed to generate business insights from an E-commerce dataset, including revenue analysis, customer analysis, country-wise performance, and monthly sales trends. The project strengthened practical knowledge of SQL and data analytics concepts.
